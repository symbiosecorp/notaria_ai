## Architecture

Modular monolith with Feature-Sliced Design adapted for TanStack. Dependency flow is strictly one-directional:

```
routes/ → modules/ → shared/ → integrations/
```

### `src/modules/<name>/`
Each domain module is self-contained with these subdirectories:
- `api/` — TanStack Start server functions (`createServerFn`) and TanStack Query `queryOptions()` factories
- `components/` — UI components scoped to this module
- `hooks/` — module-specific React hooks
- `schemas/` — Zod validation schemas
- `store/` — TanStack Store client state
- `types/` — TypeScript interfaces and types

Modules **never import from each other**. Shared logic is promoted to `shared/`.

### `src/routes/`
Thin entry points only — import from modules, do not contain business logic. Layout convention:
- `_auth.tsx` + `_auth/` — unauthenticated pages (login, register)
- `_app.tsx` + `_app/` — authenticated pages (sidebar layout)

`routeTree.gen.ts` is auto-generated by TanStack Router — never edit it manually.

### `src/integrations/`
Third-party adapters:
- `tanstack-query/root-provider.tsx` — exports `getContext()` which provides `QueryClient` to the router context
- `supabase/` — Supabase client singleton, generated DB types (`supabase gen types typescript`), SSR auth middleware
- `tanstack-ai/` — AI provider configuration

### `src/shared/`
Only code used by 2+ modules: layout components, UI primitives, global hooks, shared Zod schemas, TypeScript types.

### `src/db-collections/`
TanStack DB reactive collections (`createCollection` + `localOnlyCollectionOptions`). Schemas defined with Zod.

## Key Patterns

**Router + Query SSR integration:** `src/router.tsx` calls `setupRouterSsrQueryIntegration({ router, queryClient })`. The `QueryClient` lives in router context, accessed in routes via `Route.useRouteContext()`.

**Server functions** go in `modules/<name>/api/<name>.server.ts`:
```ts
import { createServerFn } from '@tanstack/react-start'
export const getEscritura = createServerFn({ method: 'GET' }).handler(async ({ data }) => { ... })
```

**Query options** go alongside server functions in `modules/<name>/api/<name>.queries.ts`:
```ts
import { queryOptions } from '@tanstack/react-query'
export const escrituraQueryOptions = (id: string) => queryOptions({ queryKey: ['escrituras', id], queryFn: () => getEscritura({ data: { id } }) })
```

**Route loaders** use `context.queryClient.ensureQueryData(...)` to prefetch on the server.

## Path Aliases

Both `#/*` and `@/*` resolve to `src/*` (configured in `tsconfig.json` and `package.json#imports`).

## shadcn/ui

`components.json` aliases currently point to `#/components` and `#/lib/utils`. When migrating shadcn primitives to `src/shared/components/ui/`, update `components.json` aliases accordingly before running `shadcn add`.
