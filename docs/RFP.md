# RFP — SOLICITUD DE PROPUESTA

## PROYECTO NOTARIA IA

### Sistema Integral de Gestión Notarial con Inteligencia Artificial

**Cliente:** Notaría Pública No. 1 del Distrito Judicial de Tetela de Ocampo, Puebla.

**Responsable:** Lic. Isaí Gracida Melo, Notario Auxiliar.

> 📝 *Nota manuscrita en encabezado:* **CRM / Leyes adaptadas al sistema**

---

## 1. Antecedentes

La Notaría requiere una plataforma integral que centralice la operación jurídica, administrativa, documental, fiscal y de cumplimiento normativo.

Actualmente existen procesos dispersos en Word, Excel, correo electrónico y archivos físicos.

Se busca una solución moderna, escalable y basada en inteligencia artificial que permita aumentar productividad, reducir errores y mejorar el control interno.

---

## 2. Objetivo del Proyecto

Desarrollar una plataforma web privada denominada:

### **NOTARIA IA**

Capaz de administrar integralmente:

- Clientes
- Expedientes
- Honorarios
- Instrumentos notariales
- Control documental
- UIF
- Pagos
- Agenda
- Registro Público
- Reportes
- Inteligencia Artificial Jurídica

---

## 3. Alcance General

El sistema deberá operar desde navegador web.

Deberá permitir acceso desde:

- Computadoras Windows
- Tablets
- Dispositivos móviles

El sistema deberá ser **modular**.

---

## 5. Inteligencia Artificial

La plataforma deberá incorporar **IA generativa** y **búsqueda documental**.

Funciones mínimas:

### IA Jurídica

- Identificación automática del acto jurídico
- Detección de requisitos faltantes
- Revisión de redacción
- Revisión ortográfica
- Revisión de cláusulas
- Sugerencia de fundamentos legales

### IA Documental

- Lectura de PDFs
- Lectura de imágenes
- OCR de identificaciones
- OCR de escrituras
- OCR de sentencias

Extracción automática de:

- Nombre
- CURP
- RFC
- Nacionalidad
- Estado civil
- Domicilio
- Datos registrales

### IA Operativa

- Resumen de expedientes
- Generación de listas de pendientes
- Asistente interno
- Generación de correos
- Generación de oficios
- Generación de minutas

## Visión Final

El objetivo no es desarrollar un software de expedientes, sino una **plataforma integral que funcione como sistema operativo de la Notaría**, capaz de administrar todo el ciclo de vida de una operación notarial, desde la captación del cliente hasta la firma, inscripción, archivo y generación de reportes ejecutivos.

## Anexo: Anotaciones Manuscritas

Durante la revisión del RFP físico, el responsable del proyecto agregó las siguientes anotaciones al margen:

| Sección | Anotación | Interpretación |
|---|---|---|
| Encabezado | CRM / Leyes adaptadas al sistema | Sugerencia de incorporar funcionalidad CRM y adaptación normativa local |
| Módulo Clientes | API externa → Conectada con SAT | Integración requerida con servicios del SAT para validación de RFC/CURP |
| Registro Público | Trazado de información | Necesidad de auditoría/trazabilidad sobre los movimientos registrales |
| UIF / PLD | Información del Módulo | Pendiente de ampliar especificación del módulo |
| Requisitos Técnicos (Frontend) | Vite JS | Preferencia por Vite como bundler en lugar de Next.js |
| Requisitos Técnicos (Backend) | Nest JS | Considerar Nest JS como alternativa al stack Python FastAPI |
| Seguridad | LFPDPPP · ISO 27001 | Cumplimiento obligatorio con la Ley Federal de Protección de Datos Personales y certificación ISO 27001 |
